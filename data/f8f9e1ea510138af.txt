コア機能
変更（すべてのアプリ）
ART の内部変更
Android 16 には、Android ランタイム（ART）のパフォーマンスを向上させ、追加の Java 機能のサポートを提供する、Android ランタイム（ART）の最新の更新が含まれています。Google Play システム アップデートを通じて、Android 12（API レベル 31）以降を搭載する 10 億台以上のデバイスでもこれらの改善を利用できます。これらの変更がリリースされると、ART の内部構造に依存するライブラリやアプリコードは、Android 16 を搭載するデバイスだけでなく、Google Play システム アップデートを通じて ART モジュールを更新する以前の Android バージョンでも正しく動作しなくなる可能性があります。
コア機能
変更（すべてのアプリ）
JobScheduler の割り当ての最適化
Android 16 では、アプリがどのアプリ スタンバイ バケットに属しているか、アプリが最上位の状態にある間にジョブの実行が開始されるか、フォアグラウンド サービスの実行中にジョブが実行されるかなど、いくつかの要因に基づいて、定期ジョブと緊急ジョブの実行ランタイム割り当てが調整されます。
コア機能
変更（すべてのアプリ）
空のジョブの停止理由の破棄
破棄されたジョブを検出し、減らすために、アプリは
STOP_REASON_TIMEOUT
ではなく、システムが破棄されたジョブに割り当てる新しい
STOP_REASON_TIMEOUT_ABANDONED
ジョブ停止理由を使用する必要があります。
コア機能
変更（すべてのアプリ）
順序付きブロードキャストの優先度のスコープがグローバルではなくなった
Android 16 では、異なるプロセス間で
android:priority
属性または
IntentFilter#setPriority()
を使用したブロードキャストの配信順序は保証されません。順序付きブロードキャストのブロードキャスト優先度は、すべてのシステム プロセスではなく、同じアプリケーション プロセス内でのみ尊重されます。
コア機能
変更（すべてのアプリ）
16 KB ページサイズの互換モード
Android 15 では、プラットフォームのパフォーマンスを最適化するために 16 KB メモリページのサポートが導入されました。Android 16 では、互換モードが追加され、4 KB メモリページ用にビルドされた一部のアプリを 16 KB メモリページ用に構成されたデバイスで実行できるようになりました。
コア機能
変更（16 以降をターゲットとするアプリ）
固定レートの作業スケジューリングの最適化
Android 16 以上をターゲットとするアプリの場合、アプリが有効なライフサイクルに戻ったときに、
scheduleAtFixedRate
の実行が 1 回だけ直ちに実行されます。
コア機能
新機能と API
2025 年に 2 つの Android API リリース
Android 16 では、2025 年第 2 四半期にリリースが予定されている Android の次のメジャー リリースのプレビューが提供されます。このリリースは、過去のすべての API リリースと同様に、targetSdkVersion に関連付けられることが多い計画的な動作変更が含まれています。2025 年の第 4 四半期に、新しいデベロッパー API を含む別のリリースを予定しています。2025 年の Q2 メジャー リリースは、アプリに影響を与える可能性のある計画された動作変更を含む唯一のリリースとなります。
ユーザー エクスペリエンスとシステム UI
変更（すべてのアプリ）
ユーザー補助の妨げとなるアナウンスの非推奨化
Android 16 では、
announceForAccessibility
の使用や
TYPE_ANNOUNCEMENT
ユーザー補助イベントのディスパッチを特徴とするユーザー補助のアナウンスが非推奨となります。
ユーザー エクスペリエンスとシステム UI
変更（すべてのアプリ）
3 ボタン ナビゲーションのサポート
Android 16 では、予測型「戻る」に適切に移行したアプリの 3 ボタン ナビゲーションで予測型「戻る」がサポートされます。
ユーザー エクスペリエンスとシステム UI
変更（すべてのアプリ）
テーマ別アプリアイコンの自動適用
Android 16 では、テーマがアプリアイコンに自動的に適用され、ホーム画面の統一感が向上します。
ユーザー エクスペリエンスとシステム UI
変更（16 以降をターゲットとするアプリ）
エレガントなフォント API の非推奨化と無効化
Android 16 では
elegantTextHeight
属性が非推奨となり、アプリが Android 16 をターゲットとするようになると、この属性は無視されます。
ユーザー エクスペリエンスとシステム UI
変更（16 以降をターゲットとするアプリ）
エッジ ツー エッジのオプトアウトの廃止
Android 16 以上をターゲットとするアプリの場合、
R.attr#windowOptOutEdgeToEdgeEnforcement
属性が削除されました。この属性を使用していたアプリは、ウィンドウ インセットを処理する必要があります。
ユーザー エクスペリエンスとシステム UI
変更（16 以降をターゲットとするアプリ）
予測型「戻る」には移行またはオプトアウトが必要
Android 16 をターゲットとするアプリでは、ホームに戻る、タスク間、アクティビティ間などのシステム アニメーションがデフォルトでアプリに表示されるようになりました。これをシステムに反映するため、
android:enableOnBackInvokedCallback
のデフォルト値が
true
になり、
OnBackPressed
と
KeyEvent.KEYCODE_BACK
の呼び出しは無視されるようになりました。
ユーザー エクスペリエンスとシステム UI
新機能と API
予測型「戻る」のアップデート
Android 16 では、ホームに戻るアニメーションなどのジェスチャー ナビゲーションで予測型「戻る」のシステム アニメーションを有効にするための新しい API が追加されました。Android 16 では、
finishAndRemoveTaskCallback()
と
moveTaskToBackCallback
も追加されています。
ユーザー エクスペリエンスとシステム UI
新機能と API
リッチ ハプティクス
Android 16 では、アプリがハプティクス効果の振幅と周波数の曲線を定義し、デバイスの機能の違いを抽象化できる
ハプティクス API
が追加されています。
ユーザー エクスペリエンスとシステム UI
新機能と API
進行状況を中心とした通知
Android 16 では、ユーザーが開始から終了までのユーザー主導のジャーニーをシームレスに追跡できるように、進行状況を中心とした通知が導入されています。これらの通知は、システム サーフェスでの視認性が向上し、通知ドロワーでのランキングが上位になります。
ユーザー エクスペリエンスとシステム UI
新機能と API
ライブ壁紙のコンテンツ処理
Android 16 では、ライブ壁紙フレームワークに新しいコンテンツ API が追加され、動的なユーザー主導の壁紙の課題に対応します。
セキュリティ
変更（すべてのアプリ）
インテント リダイレクト攻撃に対するセキュリティの強化
Android 16 では、
Intent
リダイレクト エクスプロイトに対するセキュリティ強化ソリューションがデフォルトで導入されています。
セキュリティ

コンパニオン アプリに検出タイムアウトが通知されなくなる
